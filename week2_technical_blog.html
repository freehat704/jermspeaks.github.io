Here are Inline-Blocks and Blocks!

Date: May 3rd, 2014

For a series of elements, particularly div and lists, you can have them display inline, inline-block, or block. This is done in CSS and uses the display property. Let’s go into detail about each of these.

Block

Block is by default for display, and for each element, the subsequent element will break a line. Take a look at the example below.

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

If we want to have this inline with each other instead of breaking, we will need a different parameter for the display. We will come back to this example later.

Inline

Say you have a bunch of paragraph tags and you wanted to combine them all into one paragraph. That's where inline is the most helpful. Take a look at this code below.

<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</p>

<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</p>

<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</p>

<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.</p>

If we wrap this in its own div, we can combine all paragraphs into one paragraphs if we assign the display to inline via CSS.

.code {display: inline;}

If you take a look at the output of this, you get the following.

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

However, if you have an image and you need it to be inline with the text, there will be issues with alignment. The images size will cause the text to have offocentered line height since it can’t have a width and height set and it will not respect top and bottom margins and padding.

As you can see, the text and the images do not respect the margins and padding. To solve this, we have inline-block.


Inline-Block

Inline-blocks respects the text between the images for the margins and padding all around. Using the same example code as above, but changing the parameter of display to inline-block, the text doesn't change its line height because of the size of the images.

.code {display: inline-block;}

Here is an example of a images with the inline-block feature. This is broken up with text after two images. Warning, hilarity ensues via Reddit's r/wheredidthesodago.

Instead of having the images as their own blocks, if left by default (the block parameter is default for display in CSS), the images are inline to each other. The text gets its own line because I separated it with a paragraph tag, but everything is now properly justified. If you want to learn other solutions to this problem, I suggest you look up flexboxes.
